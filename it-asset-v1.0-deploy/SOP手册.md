## 1. 系统概述

### 1.1 系统简介

企业运维管理系统（IT-ops）是基于 Flask + SQLite + Bootstrap 5 构建的轻量级内网运维平台，覆盖 IT 资产全生命周期管理与日常运维工作记录，适用于医院、企业等内网环境单机部署。

### 1.2 功能模块一览

| 模块 | 功能说明 | 默认权限 |
|------|----------|----------|
| **资产台账** | 设备采购、入库、分配、调拨、维修、报废全流程 | 管理员 |
| **资产分类** | 资产类别树形维护 | 管理员 |
| **软件授权** | 软件 License 管理与到期预警 | 管理员 |
| **供应商** | 供应商信息维护 | 管理员 |
| **合同管理** | 采购合同台账与到期提醒 | 管理员 |
| **域名管理** | 域名续费周期管理 | 管理员 |
| **盘点管理** | 资产盘点任务创建与执行 | 管理员 |
| **到期预警** | 汇总保修/授权/合同/域名即将到期 | 管理员 |
| **报表统计** | 资产状态、分布、折旧可视化 | 管理员 |
| **设备台账** | 运维设备（交换机/防火墙/服务器等）参数档案 | 工程师 |
| **每日巡检** | 按设备填写巡检项，记录异常与处理动作 | 工程师 |
| **工作日志** | 工程师每日工作记录，可关联设备和巡检 | 工程师 |
| **日报管理** | 当日工作汇总，可关联工作日志 | 工程师 |
| **周报管理** | 按周工作汇总 | 工程师 |
| **月报管理** | 按月工作汇总 | 工程师 |
| **用户管理** | 账户创建、权限分配（管理员专属） | 管理员 |

### 1.3 角色说明

| 角色 | 标识 | 权限范围 |
|------|------|----------|
| 系统管理员 | admin | 所有模块，含用户管理、巡检配置 |
| 运维工程师 | engineer | 由管理员按需分配具体模块权限 |

### 1.4 技术架构

```
用户浏览器 → Nginx（443/80） → Gunicorn（127.0.0.1:5003） → Flask App → SQLite
```

| 组件 | 版本 | 说明 |
|------|------|------|
| Nginx | 1.29.6 | HTTPS 反向代理，含自签名 SSL |
| Gunicorn | 最新 | WSGI 应用服务器，2 worker |
| Python/Flask | 3.6 / 2.0.3 | 应用框架（CentOS 7 兼容） |
| SQLite | 3（系统自带） | 嵌入式数据库，文件路径：`/opt/it-ops/it-asset-v1.0-deploy/asset.db` |
| Systemd | — | 服务管理，开机自启 |

## 2. 快速入门

### 2.1 访问系统

在浏览器地址栏输入：

```
https://192.168.33.165
```

> ⚠️ **自签名证书提示**：首次访问时，浏览器会提示「连接不是私密连接」，这是正常现象（内网自签名证书）。
> - **Chrome**：点击「高级」→「继续前往 192.168.33.165」
> - **Firefox**：点击「高级」→「接受风险并继续」
> - 同一浏览器后续不再提示。

### 2.2 登录

| 字段 | 说明 |
|------|------|
| 用户名 | 由管理员分配（默认管理员：`admin`） |
| 密码 | 首次使用请联系管理员获取，登录后立即修改 |

**登录安全机制**：同一 IP 在 60 秒内连续登录失败 5 次，将被临时锁定，请稍候再试。

### 2.3 修改密码

1. 登录后，点击右上角头像 / 用户名
2. 选择「修改密码」
3. 输入当前密码与新密码（新密码建议 8 位以上，含字母+数字）
4. 点击保存

> ⚠️ **重要**：管理员首次部署后请立即修改 `admin` 的默认密码。

### 2.4 退出登录

点击右上角「退出」按钮，系统自动清除 Session 并跳转到登录页。
Session 超时时间为 **8 小时**，超时后自动跳回登录页。

## 3. 用户管理

> 🔒 此模块仅管理员可访问。

### 3.1 查看用户列表

导航栏 → 「用户管理」，显示所有账户，包括：用户名、姓名、角色、权限、状态、创建时间。

### 3.2 新增用户

1. 点击「新增用户」按钮
2. 填写以下字段：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 用户名 | ✅ | 登录名，不可重复，创建后不可修改 |
| 姓名 | 推荐 | 真实姓名，用于日志展示 |
| 角色 | ✅ | `admin`（管理员）/ `engineer`（工程师） |
| 密码 | 可选 | 留空则默认 `ChangeMe123!`，首次登录后提示修改 |
| 权限 | ✅ | 勾选该用户可访问的模块（见下表） |

3. 点击「保存」

**权限模块对照表**：

| 模块标识 | 中文名 | 适合角色 |
|----------|--------|----------|
| asset | 资产台账 | 管理员/资产专员 |
| category | 资产分类 | 管理员 |
| software | 软件授权 | 管理员 |
| supplier | 供应商 | 管理员 |
| contract | 合同管理 | 管理员 |
| domain | 域名管理 | 管理员 |
| inventory | 盘点管理 | 管理员 |
| alert | 到期预警 | 管理员 |
| report | 报表统计 | 管理员 |
| equipment | 设备台账 | 运维工程师 |
| inspection | 每日巡检 | 运维工程师 |
| worklog | 工作日志 | 运维工程师 |
| report_daily | 日报管理 | 全员（默认赋予） |
| report_weekly | 周报管理 | 运维工程师 |
| report_monthly | 月报管理 | 运维工程师 |
| user | 用户管理 | 管理员 |

### 3.3 编辑用户

1. 在用户列表中找到目标用户，点击「编辑」
2. 可修改：姓名、角色、权限、状态（启用/禁用）、密码
3. 点击「保存」

> 💡 重置密码：在编辑框中输入新密码并保存，即可重置用户密码。

### 3.4 禁用/启用用户

编辑用户 → 将「状态」切换为「禁用」→ 保存。禁用后该用户无法登录。

### 3.5 删除用户

点击用户列表中的「删除」按钮。**注意**：系统默认管理员（ID=1 的 `admin` 账户）**无法删除**。

## 4. 资产管理

### 4.1 资产台账

导航 → 「资产台账」，列表展示全部资产，支持以下筛选条件：
- 资产分类
- 资产状态（库存/使用中/维修/报废/已处置）
- 使用部门
- 关键词搜索（资产编号/名称/序列号/保管人）

#### 4.1.1 新增资产（入库）

1. 点击「新增资产」
2. 必填字段：

| 字段 | 说明 |
|------|------|
| 资产名称 | 如「Dell PowerEdge R740」 |
| 资产分类 | 从分类树中选择 |
| 资产编号 | 可手动输入，留空则系统自动生成（格式：分类前缀+年月+序号） |

3. 可选字段：品牌、型号、序列号、存放地点、使用部门、保管人、采购日期、采购金额、保修到期、供应商、折旧方式、使用年限、残值率、备注
4. 点击「保存」后，系统自动创建一条「入库」操作记录

**资产状态说明**：

| 状态标识 | 中文 | 说明 |
|----------|------|------|
| in_stock | 库存 | 已入库，未分配 |
| in_use | 使用中 | 已分配给部门/人员 |
| repair | 维修中 | 送修中 |
| scrapped | 已报废 | 达到使用年限或损坏报废 |
| disposed | 已处置 | 已处置变卖或捐赠 |

#### 4.1.2 资产操作（生命周期流转）

在资产列表点击「操作」按钮，选择对应动作：

| 操作 | 说明 | 状态变更 |
|------|------|----------|
| 分配 | 分配到部门/保管人 | → 使用中 |
| 调拨 | 调换部门/地点 | → 使用中 |
| 维修 | 送修 | → 维修中 |
| 归还 | 退回入库 | → 库存 |
| 报废 | 报废处理 | → 已报废 |
| 处置 | 变卖/捐赠 | → 已处置 |

每次操作均会自动记录操作人、时间、来源和目标信息，形成完整的资产生命周期台账。

#### 4.1.3 查看资产详情

点击资产列表中的资产名称，可查看：
- 基本信息（型号、序列号、地点等）
- 折旧信息（已用月数、月折旧额、累计折旧、账面价值）
- 操作记录（入库 → 分配 → 调拨 → 维修 → ... 时间轴）

#### 4.1.4 编辑资产

在资产列表点击「编辑」，修改后保存。**资产编号不可修改**。

#### 4.1.5 删除资产

点击「删除」，同时删除该资产所有操作记录。**此操作不可逆，谨慎使用。**

### 4.2 资产分类管理

导航 → 「资产分类」，维护资产的分类树。

- **新增分类**：点击「新增」，输入分类名称，可选上级分类（留空则为顶级分类）
- **编辑分类**：点击「编辑」修改名称或层级
- **删除分类**：仅当该分类下无资产时方可删除

> 💡 建议提前规划分类体系，如：计算机设备 → 台式机/笔记本/服务器；网络设备 → 交换机/路由器/防火墙。

### 4.3 软件授权管理

导航 → 「软件授权」，管理所有软件 License。

**新增字段说明**：

| 字段 | 说明 |
|------|------|
| 授权类型 | perpetual（永久）/ subscription（订阅）/ oem |
| 授权数量 | License 总数 |
| 已用数量 | 已分配 License 数 |
| 到期日期 | 订阅类授权的到期时间，临近到期将出现在「到期预警」 |

### 4.4 供应商管理

导航 → 「供应商」，维护采购供应商信息（公司名、联系人、电话、邮箱、地址）。供应商信息关联到资产、软件授权、合同。

### 4.5 合同管理

导航 → 「合同管理」，记录采购合同台账。合同到期前 30 天将在「到期预警」中提示。

### 4.6 域名管理

导航 → 「域名管理」，管理医院/机构持有的域名续费周期。

- 域名到期前 30 天出现**橙色预警**，7 天内出现**红色预警**
- 点击「续费」可更新到期时间
- 系统实时计算剩余天数和使用进度条

### 4.7 到期预警

导航 → 「到期预警」，汇总所有**30 天内即将到期**的项目：

| 预警类型 | 数据来源 |
|----------|----------|
| 🟡 保修到期 | 资产台账（warranty_expire） |
| 🟡 软件授权到期 | 软件授权（expire_date） |
| 🟡 合同到期 | 合同管理（end_date） |
| 🟡 域名到期 | 域名管理（expire_date） |

- 🔴 **红色**：7 天内（资产/软件/合同）或 30 天内（域名）
- 🟡 **橙色**：30 天内

### 4.8 盘点管理

#### 4.8.1 创建盘点任务

1. 导航 → 「盘点管理」→「新建盘点」
2. 输入盘点名称（如「2026年6月季度盘点」）
3. 系统自动：
   - 生成盘点编号（格式：PD-YYYYMM-XXXX）
   - 快照当前所有「库存/使用中」状态的资产

#### 4.8.2 执行盘点

1. 点击盘点任务「开始盘点」
2. 逐条核对资产，点击「确认」录入实际状态
3. 不符状态的条目自动标记为「未匹配」

#### 4.8.3 完成盘点

所有资产核对完毕后，点击「完成盘点」，盘点任务状态变为 `completed`。

> 📌 本模块专为运维设备（交换机、防火墙、服务器、存储等）设计，与资产台账独立，记录技术参数。

### 5.1 查看设备列表

导航 → 「设备台账」，支持按设备类型和状态筛选，关键词可搜索名称/编号/序列号/IP地址。

**设备类型说明**：

| 类型标识 | 含义 |
|----------|------|
| switch | 交换机 |
| router | 路由器 |
| firewall | 防火墙 |
| server | 服务器 |
| storage | 存储设备 |
| ups | UPS电源 |
| other | 其他 |

### 5.2 新增设备

1. 点击「新增设备」
2. 必填：设备名称、设备类型
3. 建议填写：

| 字段 | 说明 |
|------|------|
| 设备编号 | 留空则自动生成（如 SW-202606-0001） |
| 管理IP | 设备管理地址（如 192.168.1.1） |

| 存放地点 | 机房名称/机柜编号 |

### 5.3 编辑/删除设备

- 在列表点击「编辑」可修改所有字段

## 6. 每日巡检

### 6.1 工作流程概览

```
管理员配置巡检模板 → 管理员创建巡检任务并分配工程师
→ 工程师每日执行巡检 → 工程师填写日报/工作日志关联巡检记录
```

### 6.2 巡检模板管理（管理员）

导航 → 「巡检模板」（需管理员权限）。

#### 6.2.1 新增巡检项

1. 点击「新增模板项」
2. 填写：

| 字段 | 说明 |
|------|------|
| 设备类型 | 该模板适用的设备类型（switch/firewall/server 等） |
| 巡检项名称 | 如「CPU 利用率」「端口状态」「日志告警」 |
| 巡检说明 | 如何检查的说明，显示在巡检填写界面 |
| 项目类型 | checkbox（正常/异常）/ text（填写数值）/ number（填写数字） |
| 单位 | 数值类型的单位，如 `%`、`°C` |
| 正常范围 | 如 `< 80%`，提示工程师判断基准 |
| 是否必填 | 勾选后工程师必须填写，否则无法提交 |

#### 6.2.2 排序

同一设备类型的模板项按「排列顺序」字段升序展示，可编辑该字段调整顺序。

### 6.3 巡检任务分配（管理员）

导航 → 「巡检任务」（需管理员权限）。

1. 点击「新增任务」
2. 填写：

| 字段 | 说明 |
|------|------|
| 任务名称 | 如「核心交换机日常巡检」 |
| 关联设备 | 从设备台账中选择目标设备 |
| 负责工程师 | 分配给哪位工程师执行 |
| 巡检频率 | daily（每日）/ weekly（每周一）/ monthly（每月1日） |
| 下次执行日期 | 任务生效起始日期 |
| 提醒时间 | 到点提醒工程师，默认 17:00 |

3. 保存后，系统立即为当天生成执行记录，后续按频率自动生成。

### 6.4 执行巡检（工程师）

导航 → 「每日巡检」。

1. 页面上方显示**今日我的巡检任务**列表，包含任务名称、关联设备、状态（待完成/已完成/逾期）
2. 点击「开始巡检」
3. 系统根据设备类型自动加载对应的巡检模板项
4. 逐项填写：
   - **checkbox 类型**：勾选「正常」或「异常」
   - **text/number 类型**：输入实际测量值
5. 若有异常项，在「异常描述」中填写具体情况，并在「处理动作」中描述已采取的措施
6. 填写「巡检耗时（分钟）」
7. 点击「提交」，状态变为「已完成」

> 💡 已完成的巡检记录可在工作日志中通过 ID 进行关联，形成完整的工作链路。

### 6.5 查看历史巡检

在「每日巡检」页面，可按日期、设备、工程师、状态筛选历史巡检记录，点击「查看」可展开具体巡检项明细。

## 7. 工作日志

工作日志用于记录工程师**每天的工作内容**（不限于巡检，含维修、配置变更、沟通协调等所有工作）。

### 7.1 新增工作日志

导航 → 「工作日志」→ 点击「新增」。

**填写说明**：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 日期 | ✅ | 工作日期，默认今日 |
| 工作类型 | ✅ | maintenance（维护）/ inspection（巡检）/ change（变更）/ fault（故障处理）/ other（其他） |
| 标题 | ✅ | 本条工作的简短标题，如「更换核心交换机光模块」 |
| 工作内容 | ✅ | 详细描述工作过程、发现的问题及处理结果 |
| 耗时（分钟） | 可选 | 本项工作花费的时间 |
| 关联设备 | 可选 | 从本人巡检任务涉及的设备中选择（可多选） |
| 关联巡检记录 | 可选 | 从本人当日或历史巡检记录中选择 |

> 📌 **v1.1 变更说明**：工作日志不再需要手动输入设备ID或巡检记录ID，直接从下拉菜单选择即可。

### 7.2 自动保存功能

在填写工作日志时，系统每隔 **5 秒**自动将当前表单内容保存为草稿（`draft` 状态），防止因浏览器关闭或意外操作导致内容丢失。

- 自动保存状态会在页面底部提示「已自动保存 HH:MM:SS」
- 自动保存不会触发提交流程，仅保存数据

### 7.3 提交工作日志

内容填写完毕后，点击「**提交**」按钮，状态从 `draft` 变更为 `submitted`。提交后：
- 工作日志进入可关联状态（可在日报中被引用）
- 列表中显示为「已提交」

### 7.4 编辑与删除

- 已提交的工作日志仍可编辑，点击「编辑」进行修改后重新提交
- 点击「删除」将永久删除该条记录

### 7.5 筛选查看

支持按日期范围、工作类型、关键词搜索。管理员可查看所有人的工作日志，工程师默认只看自己的。

### 10.4 数据库备份

#### 手动备份

```bash
# 方式一：直接复制数据库文件
cp /opt/it-ops/it-asset-v1.0-deploy/asset.db \
   /backup/asset_$(date +%Y%m%d_%H%M%S).db

# 方式二：打包含配置文件
tar -czf /backup/it-ops_$(date +%Y%m%d).tar.gz \
    /opt/it-ops/it-asset-v1.0-deploy/asset.db \
    /opt/it-ops/it-asset-v1.0-deploy/.env
```

#### 自动定时备份（推荐）

创建备份脚本 `/opt/it-ops/backup.sh`：

```bash
#!/bin/bash
BACKUP_DIR="/opt/it-ops/backups"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
cp /opt/it-ops/it-asset-v1.0-deploy/asset.db $BACKUP_DIR/asset_${DATE}.db
# 保留最近 30 天备份
find $BACKUP_DIR -name "asset_*.db" -mtime +30 -delete
echo "[$(date)] 备份完成: asset_${DATE}.db" >> /var/log/it-ops-backup.log
```

添加到 crontab（每天凌晨 2 点执行）：

```bash
crontab -e
# 添加：
0 2 * * * /bin/bash /opt/it-ops/backup.sh
```

### 10.6 代码更新（升级）

> ⚠️ 生产环境因无法直接访问 GitHub，代码更新需通过本地打包 + SCP 上传方式进行。

**步骤**：

```bash
# 1. 在本地开发机上打包修改的文件
# （以模板文件为例）
cd /path/to/local/it-asset-v1.0-deploy
tar -czf /tmp/update.tar.gz templates/ static/

# 2. 上传到服务器
scp /tmp/update.tar.gz root@192.168.33.165:/tmp/

# 3. 在服务器上解压并部署
ssh root@192.168.33.165
cd /opt/it-ops/it-asset-v1.0-deploy
tar -xzf /tmp/update.tar.gz
chown -R itops:itops /opt/it-ops
restorecon -R /opt/it-ops  # 修复 SELinux 上下文

# 4. 重启应用
systemctl restart it-ops
```

### 10.7 SSL 证书

当前使用自签名证书（有效期 10 年，到期时间约 2036 年），证书文件路径：

| 文件 | 路径 |
|------|------|
| 证书（公钥） | `/etc/nginx/ssl/nginx.crt` |
| 私钥 | `/etc/nginx/ssl/nginx.key` |

如需更换证书（如有域名后申请正式证书）：

```bash
# 1. 替换证书文件（保持相同路径）

chmod 644 /etc/nginx/ssl/nginx.crt
chmod 600 /etc/nginx/ssl/nginx.key

# 2. 测试并重载 Nginx
nginx -t && systemctl reload nginx
```

## 11. 常见问题排查

### 11.1 页面打开提示「连接不是私密连接」

**原因**：服务器使用自签名 SSL 证书，浏览器无法自动信任。
**处理**：
- Chrome：点击「高级」→「继续前往 192.168.33.165（不安全）」
- Firefox：点击「高级」→「接受风险并继续」
- 如需消除警告，可将 `/etc/nginx/ssl/nginx.crt` 导入到系统/浏览器受信任证书库

### 11.2 无法登录，提示「登录尝试次数过多」

**原因**：同一 IP 60 秒内连续失败 5 次，触发频率限制。
**处理**：等待 60 秒后重试，或请管理员重启应用（`systemctl restart it-ops`）清除限制。

### 11.3 系统访问报 502 错误

**常见原因**：

| 原因 | 排查命令 | 处理方式 |
|------|----------|----------|
| it-ops 服务未启动 | `systemctl status it-ops` | `systemctl start it-ops` |
| SELinux 阻止代理 | `ausearch -m avc -ts recent` | `setsebool -P httpd_can_network_connect 1` |
| Gunicorn 端口占用 | `ss -tlnp \| grep 5003` | 结束占用进程，重启服务 |

### 11.4 系统访问报 403 错误

**常见原因**：

| 原因 | 排查 | 处理 |
|------|------|------|
| 目录权限不足 | `ls -la /opt/it-ops` | `chmod 755 /opt/it-ops` |
| SELinux 上下文错误 | `ls -Z /opt/it-ops/static` | `restorecon -R /opt/it-ops` 或 `chcon -R -t httpd_sys_content_t /opt/it-ops/static` |

### 11.5 图标/样式不显示

**原因**：静态资源加载失败（权限或路径问题）。
**排查**：

```bash
# 检查静态目录权限
ls -la /opt/it-ops/it-asset-v1.0-deploy/static/
ls -la /opt/it-ops/it-asset-v1.0-deploy/static/fonts/

# 检查 SELinux 上下文
ls -Z /opt/it-ops/it-asset-v1.0-deploy/static/fonts/
```

**处理**：

```bash
chmod -R 755 /opt/it-ops/it-asset-v1.0-deploy/static/
chcon -R -t httpd_sys_content_t /opt/it-ops/it-asset-v1.0-deploy/static/
```

### 11.6 Windows Chrome 打开慢

**原因**：若静态资源路径配置错误指向外网 CDN（如 jsdelivr），内网无法访问导致超时.
**确认**：

```bash
grep -n 'jsdelivr\|cdn.bootcdn\|unpkg' \
    /opt/it-ops/it-asset-v1.0-deploy/templates/base.html
```

若有输出，说明未使用本地静态资源，请联系管理员修复。正常情况下静态资源已全部本地化，加载速度不受外网影响。

### 11.7 服务启动失败，提示数据库错误

**排查**：

```bash
# 检查数据库文件
ls -la /opt/it-ops/it-asset-v1.0-deploy/asset.db

# 验证文件是否为有效 SQLite
file /opt/it-ops/it-asset-v1.0-deploy/asset.db
```

**处理**：若数据库文件损坏，恢复最近备份（见 10.5）；若首次部署无数据库，手动初始化：

```bash
cd /opt/it-ops/it-asset-v1.0-deploy
sudo -u itops bash -c "source venv/bin/activate && \
    python3 -c 'from app import init_db; init_db()'"
systemctl restart it-ops
```

### 11.8 忘记 admin 密码

通过命令行直接重置：

```bash
cd /opt/it-ops/it-asset-v1.0-deploy
sudo -u itops bash -c "source venv/bin/activate && python3 -c \"
from models import get_db, hash_password
conn = get_db()
conn.execute('UPDATE user SET password = ? WHERE username = ?',
             (hash_password('NewPassword123!'), 'admin'))
conn.commit()
conn.close()
print('密码已重置')
\""
```
